## '기본기를 쌓을 필요가 없다'는 망상에 빠지는 것
	신입 개발자들이 AI로 만든 코드를 제대로 읽지도 못하거나, 본인이 가져온 코드가 무엇인지조차 모르는 경우가 
	허다하다고 지적합니다. 
	AI에 의존해 기본기를 포기하는 것이 평균이 되어가고 있는 위험한 상황이다.

---

### 1. 소프트웨어 설계 및 아키텍처 (Software Design & Architecture)

시스템이 복잡해져도 유지보수가 가능하고 확장성 있게 만드는 설계 능력입니다.

- **클린 아키텍처 (Clean Architecture):** 소프트웨어의 핵심 비즈니스 로직을 외부 프레임워크나 데이터베이스로부터 독립시키는 설계 방식입니다. 코드가 기술적인 세부 사항에 의존하지 않게 하여 변경에 유연하게 대응할 수 있게 합니다.
    
- **버티컬 슬라이스 아키텍처 (Vertical Slice Architecture):** 기술적 계층(Layer)이 아니라 '기능(Feature)' 단위로 코드를 나누는 방식입니다. 한 기능을 수정할 때 여러 레이어를 오가는 대신 해당 기능에 집중할 수 있게 합니다.
    
- **어댑터 패턴 (Adapter Pattern):** 서로 다른 인터페이스를 가진 클래스나 시스템이 함께 작동할 수 있도록 중간에서 변환해주는 디자인 패턴입니다. 프론트엔드에서는 외부 API와 내부 로직을 분리할 때 자주 쓰입니다.

### 2. 테스트 및 품질 보증 (Testing & Quality Assurance)

AI가 만든 코드의 결함을 찾아내고, 시스템의 안정성을 검증하는 능력입니다.

- **TDD (테스트 주도 개발, Test Driven Development):** 
  실제 코드를 짜기 전 테스트 코드를 먼저 작성하는 방식입니다.
    
    - **Red:** 실패하는 테스트 작성
        
    - **Green:** 테스트를 통과할 만큼의 최소한의 코드 작성
        
    - **Refactor:** 코드 최적화 및 구조 개선
    
- **회귀 테스트 (Regression Testing):** 
  새로운 기능을 추가하거나 코드를 수정했을 때, 기존에 잘 작동하던 기능들이 망가지지 않았는지 확인하는 필수적인 과정입니다.

### 3. 인프라 및 운영 (Infrastructure & DevOps)

코드가 배포된 이후, 실제 사용자가 몰리는 환경에서 시스템을 유지하는 기술입니다.

- **IaC (코드형 인프라, Infrastructure as Code) - Terraform:** 
  서버, 네트워크 등의 인프라 설정을 수동으로 클릭하는 대신 코드로 관리하는 방식입니다.
    
- **트래픽 처리 및 확장성 (Scalability):** 
  사용자가 급증할 때 서버가 터지지 않도록 부하를 분산(Load Balancing)하거나 서버 사양을 조절하는 전략입니다.
    
- **캐싱 전략 (Caching Strategy):** 
  자주 사용되는 데이터를 임시 저장소에 두어 응답 속도를 높이고 서버 부하를 줄이는 기술입니다.
    
- **보안 취약점 패치 (Security Patching):** 
  시스템의 보안 약점을 찾아내고 실시간으로 대응하여 개인정보와 비즈니스 데이터를 보호하는 일입니다.

### 4. 데이터 관리 (Data Management)

소프트웨어의 핵심인 데이터를 안전하고 효율적으로 다루는 방법입니다.

- **DB 마이그레이션 (Database Migration):** 
  서비스 운영 중에 데이터 구조(Schema)를 변경해야 할 때, 데이터 손실 없이 안전하게 구조를 바꾸는 전략입니다.
    
- **ETL (Extract, Transform, Load):** 
  다양한 소스에서 데이터를 추출(Extract)하고, 필요한 형태로 변환(Transform)하여, 저장소에 적재(Load)하는 데이터 처리 프로세스입니다.
    
- **비용 최적화 (Cost Optimization):** 
  클라우드 리소스 사용량을 분석하여 불필요한 비용을 줄이고 가장 효율적인 인프라 환경을 구축하는 능력입니다.
