본 보고서는 Antigravity 환경에서 Claude의 'Skills'와 유사한 생태계를 구축하고 활용하는 방법에 대해 설명합니다. 해당 기능을 통해 특정 작업에 최적화된 에이전트 능력을 프로젝트별 또는 전역적으로 설정할 수 있습니다.

---

### 1. 스킬 저장소 구조 설정

스킬을 저장할 위치는 사용 범위에 따라 두 가지 경로 중 하나를 선택합니다.

- **프로젝트 전용 (Local):** 특정 프로젝트 내에서만 해당 스킬을 활성화합니다.
    
    - 경로: `{프로젝트-루트}/.agent/skills/{스킬-이름}/`
        
- **전역 공용 (Global):** 모든 프로젝트 환경에서 공통적으로 스킬을 사용합니다.
    
    - 경로: `~/.gemini/antigravity/skills/{스킬-이름}/`
        

---

### 2. SKILL.md 작성 및 메타데이터 설정

스킬의 핵심 정의서인 `SKILL.md` 파일을 작성합니다. 파일 최상단에는 **YAML Frontmatter**를 사용하여 에이전트가 스킬의 용도를 식별할 수 있도록 정보를 입력해야 합니다.

**작성 예시:**

Markdown

```
---
name: code-reviewer-pro
description: 코드의 보안 취약점 및 성능 개선 사항을 분석하는 전문 리뷰어 스킬
usage: 코드 파일을 검토하거나 리팩토링 제안이 필요할 때 사용
---

# 스킬 상세 내용
여기에 에이전트가 수행해야 할 구체적인 프롬프트나 가이드라인을 작성합니다.
```

---

### 3. 스킬 호출 및 실행

설정이 완료되면 대화 중 스킬의 이름을 직접 언급하여 해당 기능을 실행할 수 있습니다. 에이전트는 작성된 `SKILL.md`의 메타데이터를 바탕으로 적절한 스킬을 로드하여 작업을 수행합니다.

- **실행 명령어 예시:**
    
    > "code-reviewer-pro 스킬을 사용해서 이 파일 검토해줘."
    

---

### 💡 활용 팁

- **모듈화:** 반복적으로 사용하는 프롬프트나 복잡한 지시사항을 스킬 단위로 모듈화하면 컨텍스트 효율을 높일 수 있습니다.
    
- **환경 구분:** 프로젝트마다 요구되는 코딩 컨벤션이나 보안 정책이 다를 경우, 프로젝트 전용 폴더(`.agent/skills/`)를 적극 활용하십시오.
