# AI 기반 시각적 독서 및 지식 관리 시스템

**목표:** Gemini AI를 활용해 텍스트 문서를 Obsidian Canvas(시각적 지도)로 자동 변환하여 정보 처리 속도와 이해도를 극대화합니다.

---

## 1. 개요 및 원리

- **이중 채널 이론:** 인간의 뇌는 텍스트(언어 채널)보다 이미지와 구조(시각 채널)를 동시에 사용할 때 정보를 훨씬 빠르게 처리합니다.
    
- **시스템 핵심:** Gemini CLI가 긴 텍스트를 분석하여 핵심 개념 간의 관계를 정의하고, 이를 Obsidian Canvas 파일(`.canvas`)로 자동 생성합니다.
    

## 2. 사전 준비 사항

1. **Obsidian 설치:** 최신 버전의 Obsidian 앱이 설치되어 있어야 합니다.
    
2. **Gemini API 키 발급:** [Google AI Studio](https://aistudio.google.com/)에서 무료 API 키를 발급받습니다.
    
3. **Gemini CLI 설치:** 터미널 환경에서 실행되는 Gemini 에이전트가 필요합니다. (제작자 제공 도구)
    
4. **설정 파일 다운로드:** 영상 설명란의 링크를 통해 제작자가 커스텀한 `Gemini` 설정 폴더를 다운로드합니다.
    

## 3. 초기 설정 (Setup)

1. **파일 배치:** 다운로드한 `Gemini` 폴더를 본인의 Obsidian 보관함(Vault) 최상위 루트 폴더에 복사합니다.
    
2. **환경 변수 설정:**
    
    - `Gemini` 폴더 내의 `.env.example` 파일 이름을 `.env`로 변경합니다.
        
    - 파일을 열고 `GEMINI_API_KEY=your_key_here` 부분에 발급받은 API 키를 입력합니다.
        
3. **터미널 실행:**
    
    - 터미널(또는 CMD)을 열고 Obsidian 보관함 폴더로 이동합니다.
        
    - `gemini` 명령어를 입력하여 AI를 활성화합니다.
        

## 4. 사용 방법 (Workflow)

### 단계 1: 캔버스 생성

터미널에서 다음 명령어를 입력합니다.

Bash

```
/canvas [대상파일명.md]
```

- **동작:** Gemini가 해당 마크다운 파일을 읽고, 주제별로 분류하여 시각적 노드와 화살표로 연결된 캔버스 파일을 생성합니다.
    

### 단계 2: 캔버스 확장 및 상세화

생성된 결과물이 부족하다면 AI에게 추가 요청을 할 수 있습니다.

- **명령 예시:** "방금 만든 캔버스에 구체적인 코드 예시를 추가해줘."
    
- **결과:** AI가 실시간 구글 검색 등을 활용해 정보를 보충하고 캔버스를 업데이트합니다.
    

### 단계 3: 이미지 및 미디어 처리

- 웹에서 스크랩한 기사를 시각화할 때, AI가 문서 내 이미지를 자동으로 다운로드하고 이름을 정리하여 캔버스 내 적절한 위치에 배치합니다.
    

## 5. 주요 활용 사례

- **기술 문서 정독:** 1,000단어 이상의 복잡한 개발 가이드를 한눈에 보이는 다이어그램으로 변환.
    
- **방대한 자료 요약:** 10,000단어가 넘는 논문이나 긴 보고서를 색상별로 구분된 핵심 요약 캔버스로 변환.
    
- **외국어 콘텐츠 학습:** 외국어 문서를 한국어 설명이 포함된 시각적 지도로 변환하여 이해도 증진.
    

## 6. 주의 사항 및 팁

- **파일 경로:** 모든 작업은 Obsidian 보관함 루트 디렉토리에서 터미널을 실행해야 정상 작동합니다.
    
- **프롬프트 수정:** `Gemini/prompts` 폴더 내의 파일을 수정하여 본인만의 캔버스 생성 스타일(색상, 노드 크기 등)을 커스터마이징할 수 있습니다.
    
- **검색 기능:** Gemini의 브라우징 기능을 활용하면 문서에 없는 최신 정보도 캔버스에 결합할 수 있습니다.
