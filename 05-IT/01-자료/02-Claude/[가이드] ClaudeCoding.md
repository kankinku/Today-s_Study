공유해주신 내용은 **Claude Code**를 사용할 때 매번 번거롭게 프로젝트 배경을 설명하지 않고, `CLAUDE.md` 파일을 통해 **컨텍스트를 자동으로 주입하는 최적화 방법**에 대한 가이드입니다.

주요 내용을 3가지 핵심 포인트로 요약해 드립니다.

---

### 1. 설정의 두 가지 레벨 (계층 구조)

Claude Code는 전역 설정과 프로젝트별 설정을 결합하여 인식합니다.

- **유저 레벨 (`~/.claude/CLAUDE.md`)**:
    
    - 모든 프로젝트에 공통 적용되는 **개인적인 코딩 철학**이나 **스타일**을 정의합니다.
        
    - 예: 에이전트 우선주의, TDD 지향, 이모지 사용 금지, 파일당 최대 줄 수 제한 등.
        
- **프로젝트 레벨 (`프로젝트루트/CLAUDE.md`)**:
    
    - 특정 프로젝트의 **기술 스택, 디렉토리 구조, API 규격, 컨벤션**을 정의합니다.
        
    - 예: Next.js 14 사용, 특정 라이브러리(Vitest, Tailwind) 선호, 에러 핸들링 패턴 등.
        

### 2. 효율적인 규칙 관리 (모듈화)

파일이 너무 길어지면 도메인별로 분리하여 관리할 수 있습니다.

- **규칙 분리**: `~/.claude/rules/` 폴더 내에 `security.md`, `testing.md`, `git-workflow.md` 등으로 나눕니다.
    
- **참조 방식**: `CLAUDE.md` 메인 파일에서 "상세 규칙은 특정 파일을 참고하라"는 문구만 넣어두면 Claude가 이를 파악하여 적용합니다.
    

### 3. 도입 시 기대 효과 (Before & After)

|**구분**|**기존 방식 (Prompting)**|**CLAUDE.md 활용 방식**|
|---|---|---|
|**준비 시간**|매 세션 5~10분 프로젝트 설명|설명 없이 즉시 작업 시작|
|**일관성**|대화가 길어지면 컨벤션 망각|설정된 규칙에 따라 일관된 코드 생성|
|**비용/속도**|반복적인 설명으로 토큰 낭비|토큰 50% 이상 절약 및 효율적 추론|
|**보안**|매번 보안 수칙 강조 필요|`security.md`를 통한 자동 체크|

---

> **💡 팁**: 공유해주신 내용 중 핵심은 **"Claude가 내 코딩 스타일을 이미 알고 있는 상태로 대화를 시작하게 만드는 것"**입니다. 특히 API 응답 인터페이스나 에러 처리 패턴을 미리 명시해두면 코드 리뷰나 생성 시 수정 보완 요청을 획기적으로 줄일 수 있습니다.
