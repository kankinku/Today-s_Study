#안티그래비티 #에이전트 #스킬 #생산성

---

매번 장문의 프롬프트를 입력하는 '프롬프트 노가다'에서 벗어나, 에이전트에게 **전문적인 행동 양식(Skills)**을 장착시키는 구체적인 방법론을 정리했습니다.

### 1. 왜 '스킬'인가? (프롬프트와의 차이점)

일반적인 프롬프트가 '매번 지시해야 하는 일회성 명령'이라면, **스킬**은 에이전트의 시스템에 내장되는 **'전문가 페르소나와 도구 모음'**입니다. 한 번 설정하면 에이전트가 상황에 맞춰 스스로 도구를 꺼내 씁니다.

---

### 2. 핵심 오픈소스: Antigravity Awesome Skills

GitHub의 `Antigravity Awesome Skills` 레포지토리는 에이전트의 능력을 상향 평준화해주는 130개 이상의 레시피를 제공합니다.

- **설치 대상:** Antigravity, Claude Code, Cursor 등 최신 에이전트 도구와 호환.
    
- **주요 카테고리:**
    
    - **품질 관리:** 보안 취약점 점검, 코드 린트, 테스트 케이스 자동 생성.
        
    - **성능 최적화:** 런타임 성능 분석 및 리팩토링 제안.
        
    - **프로세스 자동화:** 빌드 환경 구성 및 배포 자동화 스크립트.
        

---

### 3. 필수 장착 스킬 2종 상세 분석

이 두 가지 스킬은 에이전트의 '사고 방식' 자체를 교정하는 핵심 엔진입니다.

|필수 스킬|작동 원리|기대 효과|
|---|---|---|
|**using-superpowers**|답변 전 "이 작업에 필요한 최적의 스킬이 무엇인가?"를 먼저 자문하고 선택함.|에이전트의 섣부른 답변 방지 및 도구 활용 극대화.|
|**verification-before-completion**|"작업 완료"를 선언하기 전, 실제 빌드나 테스트 코드를 실행하여 결과를 검증함.|할루시네이션(환각) 감소 및 실행 가능한 코드 보장.|

---

### 4. 프로젝트 실전 적용 프로세스

단순히 파일을 복사하는 것을 넘어, 에이전트의 **워크플로우**에 녹여내는 방법입니다.

1. **디렉토리 구조화:** * 프로젝트 루트에 `/.agent/skills/` 폴더를 생성합니다.
    
    - 원하는 `.md` 또는 스킬 설정 파일들을 해당 폴더에 배치합니다.
        
2. **전역 규칙(Global Rules) 설정:**
    
    - 에이전트 설정(예: `.cursorrules` 또는 전역 지침)에 다음 문구를 반드시 포함합니다.
        
    
    > "모든 요청에 대해 응답하기 전, 반드시 `using-superpowers` 스킬을 호출하여 사용 가능한 스킬 목록을 확인하고 실행하라."
    
3. **지속적 재사용:** * 이렇게 설정된 스킬은 해당 프로젝트 내에서 진행되는 모든 대화 세션에 자동으로 적용됩니다.
    

---

### 5. 커스텀 스킬 제작 가이드 (SKILL.md)

자신의 비즈니스 로직이나 아키텍처 기준을 스킬로 만드는 예시입니다.

- **예시: `korean-saas-landing` 스킬**
    
    - **목적:** 한국 시장에 특화된 B2B 랜딩 페이지 생성.
        
    - **내용:** * 국내 사용자가 선호하는 폰트 크기 및 여백 기준.
        
        - '무료 체험' 대신 '도입 문의'를 강조하는 전환 유도 문구 패턴.
            
        - 신뢰감을 주는 CS/문의처 섹션 구성 필수화.
            
- **예시: `nextjs-clean-architecture` 스킬**
    
    - **목적:** 프로젝트 코드 일관성 유지.
        
    - **내용:** * `components/`, `hooks/`, `services/` 간의 의존성 규칙.
        
        - 컴포넌트 선언 시 `Arrow Function` 사용 강제.
            
        - API 호출부의 에러 핸들링 표준화.
            

---

> **핵심 결론:** 훌륭한 엔지니어는 프롬프트를 잘 쓰는 사람이 아니라, 에이전트가 알아서 잘 일하도록 **환경(Skills)**을 잘 구축하는 사람입니다.
