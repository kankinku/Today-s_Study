**(가칭: FlowBid)**

#### **1. 아이디어 한줄 정의**

> **"기업이나 개인이 해결하고 싶은 '사소하지만 귀찮은 디지털 업무'를 자연어로 입력하면, AI 에이전트와 인간 빌더가 경쟁/협력하여 10분 내에 '일회용 마이크로 SaaS'를 생성해 납품하고 즉시 결제받는 실시간 마켓플레이스."**

---

#### **2. 왜 지금 반드시 되는가 (2026년 2월 타이밍)**

- **기술적 필연성 (MCP & Vibe Coding):** 2026년 현재, **MCP(Model Context Protocol)**의 보편화로 AI가 단순히 텍스트를 생성하는 것을 넘어, 사용자의 로컬 파일, DB, 슬랙, 이메일 시스템과 안전하게 연결됩니다. 또한 **바이브 코딩(Vibe Coding)** 덕분에 자연어만으로 복잡한 UI와 로직을 짤 수 있어, '코딩'이 더 이상 진입장벽이 아닙니다.
- **시장 변화 (소프트웨어의 노동화):** a16z가 예측했듯, 소프트웨어는 자본재에서 노동재로 변환되었습니다. 사람들은 거창한 ERP를 도입하는 게 아니라, 당장 "오늘 들어온 송장 50개만 정리해줘"라는 구체적이고 일회성인 노동을 수행할 소프트웨어를 원합니다.
- **롱테일 문제 해결:** 기존 SaaS는 대중적 기능에 집중하느라 수만 가지의 **롱테일(Long-tail)** 니즈(예: 특정 형식을 특정 엑셀로 변환)를 무시했습니다. AI는 이 무한한 틈새시장을 비용 효율적으로 메울 수 있는 유일한 도구입니다.

---

#### **3. 돈이 어떻게 들어오는가 (수익 구조)**

- **즉시 매출 (Bounty Model):** 요청자가 "이 엑셀 파일 정리해주면 $10 냄"이라고 공고(Bounty)를 올립니다.
    - **AI 해결:** AI가 즉시 해결하면 플랫폼이 수수료 50%를 가져갑니다.
    - **인간 해결 (Vibe Coder):** AI가 실패한 복잡한 문제를 인간이 바이브 코딩으로 해결하면, 해결사가 80%, 플랫폼이 20%를 가져갑니다.
- **재판매 로열티 (Passive Income):** 한 번 만들어진 해결 도구(Workflow)는 라이브러리에 저장됩니다. 다른 사용자가 같은 도구를 유료로 사용할 때마다 원작자(또는 프롬프트 엔지니어)에게 로열티가 지급됩니다. 이는 '1,000명의 진정한 팬'을 모으는 크리에이터 경제를 소프트웨어 개발 영역으로 확장합니다.

#### **4. MVP 설계 (1인 개발 & 바이브 코딩 기준)**

**[핵심 철학]** "기능이 아니라 **가치 가설(Value Hypothesis)**을 검증하는 최소한의 제품"

- **필요한 기능 (5개 이하):**
    
    1. **Request Input (Natural Language):** "무엇을 해결하고 싶으신가요?" (예: "이 PDF에서 표만 추출해서 내 구글 시트에 넣어줘").
    2. **Auto-Generator (Agent):** LLM + MCP를 활용해 요청을 분석하고, 미리 정의된 **디자인 시스템(Atomic Design)**을 조립하여 즉시 실행 가능한 웹 도구 생성.
    3. **Preview & Run:** 생성된 도구를 웹상에서 바로 실행(Sandbox 환경).
    4. **Escrow Payment:** 결과물 확인 후 결제 승인.
    5. **Remix:** 기존 솔루션을 복제하여 수정하는 기능.
- **사용 기술 스택 (2026년 기준 1인 최적화):**
    
    - **Frontend:** Next.js + 제공된 **디자인 시스템 가이드** (Tailwind CSS 기반의 Token System 적용).
    - **AI/Logic:** GPT-5 또는 Claude 4 (가정) API + LangChain (Agent Orchestration).
    - **Database:** Supabase (벡터 DB 포함).
    - **Design System:** `tokens.json`을 활용하여 다크모드/반응형 UI 자동 대응.
- **Vibe Coding & AI 활용 방식:**
    
    - 사용자가 요청하면 AI는 **디자인 토큰** (`--color-primary-600`, `--radius-md` 등)을 사용하여 UI를 실시간으로 렌더링합니다.
    - 복잡한 로직은 MCP 서버를 통해 사용자의 로컬 데이터나 외부 API와 안전하게 연동합니다.

#### **5. 성장 폭발 메커니즘 (Growth Engine)**

- **바이럴 엔진 (Viral Engine of Growth):**
    - 문제를 해결한 사용자가 결과물(자동화된 툴)을 동료에게 공유합니다. "이거 써봐, 1초 만에 끝나."
    - 동료는 해당 툴을 사용하기 위해 가입하며, 자신도 필요한 툴을 요청하게 됩니다. (바이럴 계수 K > 1 목표).
- **네트워크 효과:**
    - 해결된 문제(솔루션)가 데이터베이스에 쌓일수록, AI는 더 똑똑해지고 해결 속도는 빨라집니다. 후발 주자가 따라올 수 없는 **'해결된 워크플로우 데이터'**가 쌓입니다.
- **락인(Lock-in) 전략 - 1,000 True Fans:**
    - 바이브 코딩 능력이 좋은 '해결사(Creator)'들이 플랫폼 내에서 활동하며 수익을 창출합니다. 이들은 자신의 팬(의뢰자)을 관리하며 플랫폼을 떠나지 못하게 만드는 앵커 역할을 합니다.

#### **6. 1인 → 유니콘 시나리오**

- **Step 1: 0 → 월 $1k (컨시어지 MVP)**
    - **전략:** 자동화 코딩을 구현하지 않습니다. 랜딩 페이지만 만듭니다.
    - **실행:** 특정 커뮤니티(예: 온라인 셀러)에 "반복 작업 대신 해드립니다" 홍보. 주문이 들어오면 **운영자가 직접** LLM을 써서 코드를 짜고 배포해줍니다. 고객은 자동화된 줄 알지만 실제로는 사람이 하는 **Wizard of Oz** 방식입니다.
- **Step 2: $1k → $100k (자동화 & 스티키 엔진)**
    - **전략:** 자주 들어오는 요청(엑셀 변환, 크롤링 등)을 템플릿화하여 AI가 자동 생성하게 만듭니다.
    - **지표:** 재구매율(Retention)에 집중합니다. 한 번 쓴 사람이 또 쓰는지 확인합니다.
- **Step 3: $100k → $10M+ (플랫폼 확장)**
    - **전략:** 기업용 'Private Agent Store'로 확장합니다. 기업 내부의 보안 데이터를 다루는 MCP 기반 사내 툴 마켓으로 피벗(Pivot)합니다.
    - **인력 유지:** AI가 대부분의 코딩과 CS를 처리하므로, 핵심 기획과 고도화 전략 인력만 소수로 유지합니다.

#### **7. 경쟁자가 생겨도 이기는 구조 (해자 - Moat)**

- **독점적 학습 데이터:** 단순히 코드를 짜주는 AI는 많지만, **"실무자들이 현업에서 어떤 구체적인 디테일(Edge case) 때문에 고통받는지"**에 대한 데이터는 FlowBid에만 쌓입니다. 이 데이터가 쌓일수록 AI의 해결률이 압도적으로 높아집니다.
- **디자인 시스템의 표준화:** 제공된 디자인 가이드에 따라 **접근성(WCAG AA)**과 **다크모드**가 완벽히 지원되는 고품질 UI가 자동 생성되므로, 조잡한 경쟁사 툴보다 사용자 경험이 월등합니다.

#### **8. 바로 검증하는 방법 (Action Plan)**

> **오늘 당장 실행 가능한 검증 단계 (자본 $0)**

1. **가설 설정:** "사람들은 자신의 사소한 반복 업무를 해결해주는 웹 도구에 $5를 지불할 의사가 있다." (가치 가설).
2. **스모크 테스트 (Smoke Test):**
    - 간단한 랜딩 페이지를 만듭니다. "당신의 엑셀/PDF 반복 업무, 10분 만에 자동화 툴로 만들어 드립니다. 건당 $5."
    - 결제 버튼을 누르면 "현재 주문 폭주로 대기 명단에 등록됩니다"라고 띄우고 이메일을 수집합니다.
3. **트래픽 유도:**
    - Reddit, Twitter, 혹은 특정 직군 커뮤니티에 "내가 만든 툴로 업무 시간 2시간 줄였다"는 식의 바이럴 게시글을 올립니다.
4. **검증 기준:**
    - 방문자 대비 이메일 등록률(Conversion Rate)이 10% 이상이면 개발 착수. 그 이하면 타겟이나 메시지 변경(Pivot).
